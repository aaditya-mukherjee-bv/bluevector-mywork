Frequently Asked Questions

Find answers to common questions about Agentic App Development and the SnapApp™ platform.

What is the difference between an "Event" and a "Timing" trigger?
expand_more

An Event trigger is reactive; it fires the moment something happens in the system (e.g., a "Submit" button is clicked). A Timing trigger is proactive; it fires based on a clock or a calendar (e.g., "Send a reminder email 3 days before a permit expires"). SnapApp allows you to combine both for highly sophisticated workflows.

Can we build multi-department workflows?
expand_more

Absolutely. SnapApp’s engine is designed for cross-functional collaboration. You can build a workflow that starts in one department (e.g., Intake), moves to another for technical review (e.g., Engineering), and concludes in a third for final approval (e.g., Finance), all while maintaining a single, auditable record.

How does the system handle "stuck" workflows?
expand_more

We include built-in Exception Handling. If a task sits idle for too long or if an automated step encounters an error, the system can automatically escalate the issue to a supervisor or trigger a "fallback" path to ensure the process never grinds to a halt.

Is this HIPAA and FedRAMP compliant?
expand_more

Yes. Because the engine runs on Google Cloud Platform and is governed by SnapApp’s core security layers, every automated task inherits the same high standards of encryption, data residency, and access control required for sensitive Government and Healthcare data.

Do I need to be a developer to build these workflows?
expand_more

No. The Workflow Designer is a visual, "drag-and-drop" interface. While it supports advanced technical configurations (like webhooks), the primary logic is built using a clear, graphical map that Business Analysts and Department Leads can easily understand and manage.
